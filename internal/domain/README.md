# Domain Layer

Доменный слой описывает предметную область без зависимостей на HTTP, SQL, ORM и внешние API.

## Контексты

- `identity`:
  - пользователь, часовой пояс, цель шагов, streak.
- `steps`:
  - дневные шаги, источники данных, week/month аналитика.
- `medication`:
  - карточка БАД, расписание, логи приема, переходы статусов.
- `workout`:
  - агрегат выполнения тренировки (`Workout`) и агрегат шаблона (`TrainingTemplate`).
- `dashboard`:
  - read-model главного экрана.
- `shared`:
  - базовые value objects и ошибки домена.

## Принципы

- Валидация инвариантов выполняется в конструкторах/методах домена.
- Интерфейсы репозиториев объявлены в домене, реализации находятся в инфраструктуре.
- Время передается в домен явно (`time.Time`/`Clock`), чтобы тесты были детерминированными.
